# 小红书"最热"文章定义

## 热门算法维度

小红书的热门推荐算法基于多个维度综合计算：

### 1. 核心互动指标
- **点赞数**（likes）：用户对内容的认可度
- **收藏数**（collects）：内容的实用价值
- **评论数**（comments）：话题讨论热度
- **分享数**（shares）：内容的传播力

### 2. 时间衰减因子
- 发布时间越近，权重越高
- 通常 24-72 小时内的内容有热度加成

### 3. 热门计算公式（推测）
```
热度分数 = (点赞数 × 1.0 + 收藏数 × 1.5 + 评论数 × 2.0 + 分享数 × 3.0) 
          / (发布时间的小时数 + 1)^0.8
```

**收藏权重高**：因为小红书是种草平台，收藏代表购买意向
**分享权重最高**：代表内容传播价值

### 4. 平台热门榜单

#### A. 综合热门（发现页）
- 算法推荐的优质内容
- 基于用户兴趣个性化

#### B. 搜索热门
- 特定关键词的搜索结果排序
- 按"最新"或"最热"排序

#### C. 话题热门
- 特定话题标签下的热门笔记
- 按互动数据排序

## 实际抓取策略

### 策略 1：关键词搜索 + 排序
```
搜索关键词 "AI" 
→ 按"最热"排序
→ 取前 20-50 条
→ 筛选互动数据 > 阈值
```

### 策略 2：话题热门
```
访问话题页 #AI 
→ 获取热门标签
→ 提取高互动笔记
```

### 策略 3：综合评分过滤
```python
def calculate_hot_score(note):
    """
    计算热度分数
    """
    likes = note.get('likes', 0)
    collects = note.get('collects', 0)
    comments = note.get('comments', 0)
    shares = note.get('shares', 0)
    hours = note.get('publish_hours', 24)
    
    score = (likes * 1.0 + collects * 1.5 + 
             comments * 2.0 + shares * 3.0) / (hours + 1) ** 0.8
    
    return score

# 热门阈值（根据类目调整）
```

## 最佳实践

1. **每天定时抓取**：凌晨 2-3 点（流量低峰）
2. **多关键词组合**：`["AI", "人工智能", "ChatGPT", "AIGC"]`
3. **去重机制**：基于笔记 ID 去重
4. **增量更新**：只抓取新发布的笔记
5. **热度监控**：追踪热度变化趋势
