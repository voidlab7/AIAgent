# 📊 Trend Analyst — 选题分析 Skill

> 从热点数据到选题报告的全流程自动化分析工具

## 触发条件

当用户提到以下关键词或场景时使用此 Skill：

**直接关键词**：
- `选题分析` / `选题报告` / `热点分析` / `趋势分析`
- `爆款选题` / `热门选题` / `话题分析`
- `小红书分析` / `小红书选题` / `小红书热点`
- `趣味测试分析` / `心理测试分析` / `MBTI分析`

**场景触发**：
- 用户说"帮我分析一下XX在小红书上的热度"
- 用户说"XX这个选题值不值得做"
- 用户说"帮我调研一下XX方向"
- 用户描述一个产品 IDEA，想了解市场数据
- 用户想做一个新的网站/小程序，需要先做选题验证

**不应触发的场景**：
- 用户只是在讨论已完成的报告内容（不需要重新分析）
- 用户在做具体的 UI 设计或写代码（应由设计/开发 Skill 处理）
- 纯粹的写作创作需求（不涉及数据分析）

---

## 核心能力

这个 Skill 帮助用户完成从"一个模糊的产品想法"到"一份有数据支撑的选题分析报告"的全流程。

### 它能做什么
1. **关键词拓展**：基于用户的主题，拓展出 3-5 个高覆盖搜索关键词
2. **数据采集**：调用小红书 MCP 抓取各关键词下的热门帖子
3. **深度分析**：获取 TOP 帖子的详情和评论，挖掘用户真实需求
4. **报告输出**：生成结构化的选题分析报告，包含数据、洞察和行动建议

---

## 执行流程

### Phase 1：理解需求 & 关键词拓展

**目标**：把用户模糊的想法转化为可执行的搜索策略

1. 确认分析主题（用户想分析什么方向？）
2. 拓展搜索关键词：
   - 主关键词（用户直接提到的）
   - 同义词/变体词（换个说法搜同一个东西）
   - 上位词/下位词（更宽泛或更细分的搜索）
   - 用户视角词（用户在搜索这类内容时会用什么词）
3. 输出关键词列表（3-5 个），和用户确认

**关键词拓展示例**：
```
用户输入：宠物MBTI测试
拓展结果：
- 宠物性格测试（同义变体）
- 猫咪性格测试（下位词 - 猫咪是主流）
- 狗狗MBTI（下位词 - 狗狗细分）
- 宠物MBTI（原始词）
- 猫咪MBTI（下位词 - 高搜索量）
```

### Phase 2：数据采集

**目标**：通过小红书 MCP 获取足够的数据样本

#### 步骤 2.1：批量搜索
对每个关键词，调用小红书 MCP 的 `search_feeds` 工具：

```
工具: search_feeds
参数: keyword="关键词"
获取: 每个关键词 20 条结果
```

记录每条笔记的核心数据：
- 标题、作者
- 点赞数、收藏数、评论数、分享数
- 笔记类型（图文/视频）
- 笔记 ID（用于后续详情获取）

#### 步骤 2.2：获取 TOP 帖子详情
从搜索结果中筛选 TOP 5-10 的高互动帖子，调用 `get_feed_detail` 获取：

```
工具: get_feed_detail
参数: feed_id="笔记ID"
获取: 完整内容、评论列表
```

重点关注：
- 帖子的完整文案（分析内容结构和卖点）
- 评论区热门评论（挖掘用户真实需求和痛点）
- 评论区是否出现"在哪里XX"、"求链接"等需求信号

#### 去重规则
同一笔记 ID 只保留一次，跨关键词搜索结果可能有重叠。

### Phase 3：数据分析

**目标**：从原始数据中提炼出有价值的洞察

#### 3.1 数据总览
统计维度：
| 指标 | 说明 |
|------|------|
| 总帖子数 | 去重后的帖子数量 |
| 总点赞/收藏/评论/分享 | 各指标汇总 |
| 平均互动数据 | 每篇帖子的平均数据 |
| 图文 vs 视频占比 | 哪种形式更受欢迎 |

#### 3.2 TOP 帖子排行
按热度公式排序（权重：点赞×1 + 收藏×1.5 + 评论×2 + 分享×3），列出 TOP 10。

#### 3.3 内容类型分类
根据标题和内容关键词，将帖子分成 3-6 种内容类型，每种类型统计：
- 帖子数量
- 平均互动数据
- 代表作品（TOP 3）
- 内容公式总结

类型分类要基于实际数据，不要预设。观察标题和内容中的共性模式，归纳出类型。

#### 3.4 评论区洞察
从 TOP 帖子的评论中提取：
- 用户最关心的话题
- 高频出现的需求词（"在哪里测"、"求教程"等 → 产品机会）
- 用户的情绪倾向（正面/负面/中性）
- 社交互动模式（@好友、评论区集合等）

#### 3.5 关键指标分析
特别关注这些"超常指标"：
- **评论/赞比 > 20%**：说明互动性极强
- **分享/赞比 > 50%**：说明社交裂变属性强
- **收藏/赞比 > 30%**：说明实用/干货价值高

### Phase 4：报告生成

**目标**：输出一份完整的选题分析报告

报告输出到 `产品IDEA/` 目录，命名格式：`{序号}、{主题名}.md`

#### 报告结构模板


### Phase 5：下一步建议

分析报告完成后，主动给出下一步行动建议：

1. **如果是验证产品 IDEA**：建议是否值得做，优先做什么
2. **如果是内容创作**：推荐具体的选题和创作模板
3. **如果需要深入**：建议是否需要做设计文档（引导到 product-designer Skill）

## 质量标准

一份好的选题分析报告应该满足：

1. **数据充分**：至少覆盖 3 个搜索关键词，总采样量 50+ 条帖子
2. **洞察有价值**：不是简单罗列数据，而是提炼出可执行的结论
3. **评论区深挖**：TOP 帖子的评论是最有价值的需求信号来源
4. **有行动建议**：报告最后要有明确的"下一步做什么"
5. **格式统一**：遵循 `产品IDEA/` 目录下已有报告的风格

## 常见问题

### Q: 小红书 MCP 未登录怎么办？
先调用 `check_login_status` 检查状态，如果未登录，引导用户通过 `get_login_qrcode` 扫码登录。

### Q: 搜索结果太少怎么办？
1. 尝试更宽泛的关键词
2. 减少搜索关键词的专业性（用用户常用的口语化表达）
3. 增加关键词数量（从 3 个扩展到 5-7 个）

### Q: 如何判断一个选题值不值得做？
关键看三个信号：
- **互动数据高**：TOP 帖子平均点赞 > 1000
- **评论区有需求缺口**：反复出现"在哪里XX"、"求推荐"
- **竞品少或质量低**：现有内容形式有明显改进空间
